
## Metrics
Metrics are used to calculate progression values such as totals, percentages, and progress bars.  
They allow you to convert raw data like levels, upgrades, or milestones into formatted results that can be displayed in menus, holograms, scoreboards, and other interfaces.

Metrics are built using three parts:

- **Sources** – provide the input data
- **Calculations** – process the data
- **Results** – format the output

<br>

### Syntax
```yaml
metrics:

 - id: METRIC_ID
   sources: SOURCES
   calculations: CALCULATIONS
   results: RESULTS
```

<br>

### id
```yaml
id: METRIC_ID
```

Sets the unique identifier of the metric.

This identifier is used when referencing the metric in placeholders.

<br>

### sources
```yaml
sources: SOURCES
```

Sets the input values used by the metric.

Sources provide the raw data required to perform calculations.  
They can represent things such as:

- player progression
- maximum progression values
- fixed reward values
- external data providers

Sources can be referenced inside calculations.

<br>Check the **Sources** section for more information.

<br>

### calculations
```yaml
calculations: CALCULATIONS
```

Sets the mathematical expressions used to process the source values.

Calculations can reference:

- sources
- other calculations

They are typically used to calculate things such as:

- current progression
- maximum progression
- remaining progression
- completion percentage

<br>Check the **Calculations** section for more information.

<br>

### results
```yaml
results: RESULTS
```

Sets the formatted outputs generated by the metric.

Results convert calculation values into display-ready outputs such as:

- numbers
- formatted text
- percentages
- progress bars

These results can be referenced through placeholders and used in menus or other interfaces.

<br>Check the **Results** section for more information.

<br>

### Result Placeholders

Metric results can be accessed using placeholders.

#### DEDI Placeholder
```
{mtr--<fileID>_<metricID>_<resultID>}
```

#### PlaceholderAPI
```
%smccore_mtr--<fileID>_<metricID>_<resultID>%
```

<br>

### Flow Overview

Metrics follow a simple processing flow:

```
Sources → Calculations → Results
```

1. **Sources** provide the raw data.
2. **Calculations** process the data.
3. **Results** format the output.

This structure allows metrics to calculate progression for many systems such as:

- collections
- skills
- upgrades
- achievements
- tasks
